# PK/PD Nexus AI · 2030 UX Direction

This demo should feel like a 2030 clinical operations product, not a technical showcase.

## Product feeling

- Calm
- Fast
- Predictive
- Agentic
- Invisible interoperability

The user should feel that the system has already done meaningful work before they click anything.

## What the UI should communicate

1. The important cases rise automatically.
2. The patient story arrives already assembled.
3. The next best move is already being prepared.
4. The right expert is brought in at the right time.
5. Every validated decision makes the next case easier.

## What the UI should hide

- Raw infrastructure language
- Academic wording
- Technical stack explanations
- Backend terminology like FHIR, RAG, vector search, deterministic retrieval, unless needed in a secondary technical layer

## Demo voice

Prefer:

- “Patient story”
- “Recommended playbook”
- “Cases that need attention now”
- “Ready-to-send handoff”
- “What experts did in similar situations”
- “Reusable team memory”

Avoid as primary visible copy:

- “FHIR backbone”
- “Semantic retrieval”
- “Knowledge products”
- “Deterministic priority factors”
- “Protocol retrieval”

## 2030 behaviors we want to imply

- Automation before interaction
- AI drafting inside safe human workflows
- Invisible interoperability
- Context that follows the user across desktop and mobile
- Human review as the last mile, not the first step

## Current UX upgrades already aligned with this

- More outcome-first copy across landing, desktop, and mobile
- Less technical provider labeling
- A visible “already in motion” automation strip in the live case view
- More human-readable section naming in `/pro`

## Highest-value remaining polish

1. Make the hero case flow deterministic every time.
2. Upgrade the copilot to feel more like an action orchestrator than a text generator.
3. Replace at least one placeholder evidence source with real official content.
4. Add more elegant loading, empty, and transition states for live demo use.
